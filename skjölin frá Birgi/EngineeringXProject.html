Datamart for Icelandic pension fund data

Prepared for: Symmetria ehf.

Contact person: Birgir Arnarson, birgir.arnarson@symmetria.is

1. The aim of the project

The goal of the project is to make data on Icelandic pensions funds accessible via a relational database. The Icelandic pension fund system is among the best in the world. People contribute part of their salaries into the pension funds that in turn invest the constributions over a long time before paying it back to the members in the form of old age benefits, disability benefits, spouse benefits and child benefits. The benefits depend on the contributions, together with long term investments and returns, and it can be hard to comprehend the dynamics that leads to the final outcome for a given member. Understanding this long term dynamics requires access to longitudinal data on the pension funds, and the best way is to complie the data in one place in a relational database. The data, which includes information on pension fund assets, liabilities, various member attributes, cash flows, investments, etc., comes in a variety of formats. The project revolves around so called Extract-Transform-Load (ETL) processes. The database would serve as a useful tool for analysis of the Icelandic pension fund system in the context of academic research as well as for regulatory, policy, and commercial purposes.

2. Detailed description

The data exists and is readily available in the form of excel sheets, PDF documents and webpages. The supervisor will provide a list of the primary data entities and attributes that are required to be extracted. The main tasks are as follows:

  1. Design and implement a relational database structure for the data. The structure will be based on something resembling a snowflake.

  2. Design and implement processes to extract data from excel sheets, PDF documents and webpages and save the data into the data structure in 1., with a possible intermediate staging area in the database. This will involve one or more of the following software tools: Python/Julia, RegEx, SQL, etc. Some of the older data may require special solutions, but the more recent data should be extracted by tools that can be used to maintain the database going forward.

If time permits, applications of the database might be implemented. The database can be used, e.g., in the context of risk management and agent based simulations of pension funds and the financial system as a whole.

3. The outcome of the project

The outcome of the project is a relational database containing historical information on the Icelandic pension fund system. The database is implemented in PostGres, an open source database system, and will be open to researchers and others who need to work with the data. The software tools will be publicly available via github.

4. The scope of the project

Completing the project might take approximately 200 man hours per person plus the time needed to get acquainted with the necessary tools. Time may not be sufficient to complete the project, but partial completion would still be valuable to the company. The students will get a taste of the real world and learn how to use modern software tools to extract and work with data. The time invested to understand the tools and the ETL processes is well spent and will greatly benefit the students in the future.

5. Facilities and workspace

Students will use their own computers and RU facilities. Meetings with the supervisor will take place in RU.

6. Student qualifications

The primary requirement for a student is interest in learning ETL processes and the underlying tools. It would be beneficial for the students to be familar with programming languages such as Python or Julia, and familiarity with SQL and databases is also helpful. However, neither is required and the supervisor will provide guidance on implementing and learning the necessary skills to complete the project.
